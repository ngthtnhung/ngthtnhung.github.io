Project detail

Product Management DevOps Platform

Node.js + Express product management application with MongoDB persistence, fault-tolerant in-memory fallback, file uploads, Prometheus monitoring, Docker Swarm orchestration, and Terraform infrastructure automation.

Overview

This project is a Node.js and Express product management application built for a DevOps final assignment. It provides a server-rendered web interface and a REST API, with robust deployment and monitoring support for production-like infrastructure.

Key features

  • Product catalog UI rendered with EJS
  • REST API for product CRUD operations
  • MongoDB persistence through Mongoose
  • In-memory fallback data source with seeded Apple product data
  • Product image uploads via Multer
  • Prometheus metrics endpoint at /metrics
  • Docker image build support and Docker Swarm orchestration
  • Terraform and server setup scripts for infrastructure provisioning
  • CI/CD via GitHub Actions

Technical stack

  • Runtime: Node.js 20
  • Backend: Express.js
  • Views: EJS
  • Database: MongoDB with Mongoose
  • Validation: express-validator
  • Uploads: Multer
  • Metrics: prom-client, Prometheus, Grafana, Alertmanager
  • Containerization: Docker
  • Orchestration: Docker Swarm
  • Reverse proxy: Nginx
  • Infrastructure: Terraform
  • CI/CD: GitHub Actions

Local setup

Install dependencies and start locally:

npm install
cp .env.example .env
# On Windows PowerShell:
Copy-Item .env.example .env
npm start
# For development with auto-reload:
npm run dev

If MongoDB is reachable within 3 seconds, the app uses MongoDB. Otherwise it falls back to an in-memory data source so the UI and API remain available.

API endpoints

Base URL: http://localhost:3000/products

  • GET /products — list all products
  • GET /products/:id — get one product by ID
  • POST /products — create a product
  • PUT /products/:id — replace a product
  • PATCH /products/:id — partially update a product
  • DELETE /products/:id — delete a product

Example fields include name, price, color, description, imageUrl, and optional multipart upload field imageFile.

Metrics

The application exposes Prometheus metrics at /metrics, including default Node.js process metrics and a custom HTTP request duration histogram named http_request_duration_seconds.

Outcome

This project demonstrates a resilient web application architecture that remains available even when the database is unavailable, supports modern monitoring and container orchestration, and includes automation for production-grade infrastructure.
